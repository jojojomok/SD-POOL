# 产品经理需求管理工具 - 设计规格文档

## 1. 概述

产品经理日常待办（需求）管理工具，用于 PM、上级、需求方三方协作，管理需求池中所有需求的完整生命周期。

## 2. 技术栈

| 组件 | 技术选型 |
|------|----------|
| 框架 | Next.js 14 (App Router) |
| 样式 | Tailwind CSS |
| 数据库 | Supabase (PostgreSQL) |
| 认证 | NextAuth.js / Supabase Auth |
| 部署 | Vercel |
| 定时任务 | Vercel Cron Jobs |

## 5. 需求生命周期

### 5.1 四个阶段与十三个状态

**阶段一：需求提出 - PM跟进**
1. 待跟进
2. 跟进中

**阶段二：评审 & 报价**
3. 待审批
4. 待用户确认
5. 开发评审
6. 报价中
7. 待确认华定币

**阶段三：开发 & 测试**
8. 待开发
9. 开发中
10. 测试中

**阶段四：验收 & 上线**
11. 用户验收
12. 待发布
13. 已上线

### 5.2 状态流转规则

- 只允许**前进**：可顺序流转，也可跳转（如从"跟进中"直接到"报价中"）
- **不支持回退**
- 阶段由状态**自动匹配**，无需手动选择

### 5.3 状态→阶段映射

| 阶段 | 包含状态 |
|------|----------|
| 阶段一：需求提出 - PM跟进 | 待跟进, 跟进中 |
| 阶段二：评审 & 报价 | 待审批, 待用户确认, 开发评审, 报价中, 待确认华定币 |
| 阶段三：开发 & 测试 | 待开发, 开发中, 测试中 |
| 阶段四：验收 & 上线 | 用户验收, 待发布, 已上线 |

## 6. 数据模型

### 6.1 需求表 (requirements)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| title | TEXT | 需求标题 |
| brief | TEXT | 需求 brief |
| description | TEXT | 需求描述 |
| prd_url | TEXT | PRD 链接 |
| mockup_url | TEXT | 墨刀链接 |
| submitter_id | UUID | 提出人（关联用户表） |
| system | TEXT[] | 所属系统（多选：HIFOOD 1.0, HIFOOD 2.0, SAP, CRM, AI） |
| module | TEXT | 所属模块 |
| priority | ENUM | P0 / P1 / P2 / P3 / P4 |
| deadline | DATE | 期望完成时间 |
| estimate | NUMERIC | 预估工时/报价 |
| assignee_id | UUID | 负责人（关联用户表） |
| version | TEXT | 所属迭代/版本 |
| status | TEXT | 当前状态（13个之一） |
| phase | TEXT | 自动计算（根据 status 映射） |
| attachments | JSONB | 附件链接列表 |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间 |

### 6.2 用户表 (users)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | TEXT | 姓名 |
| email | TEXT | 登录邮箱 |
| role | ENUM | pm / boss / stakeholder |
| department | TEXT | 部门（需求方用） |

### 6.3 评论表 (comments)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| requirement_id | UUID | 关联需求 |
| user_id | UUID | 评论人 |
| content | TEXT | 评论内容 |
| created_at | TIMESTAMPTZ | 时间 |

### 6.4 操作日志表 (logs)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| requirement_id | UUID | 关联需求 |
| user_id | UUID | 操作人 |
| action | TEXT | 操作类型（状态变更、字段修改等） |
| detail | JSONB | 变更详情（改前改后） |
| created_at | TIMESTAMPTZ | 时间 |

## 7. 页面设计

### 7.1 需求看板（首页）
- 四列看板，对应四个阶段
- 每个阶段列中展示处于该阶段的需求卡片
- 顶部工具栏：搜索、筛选（系统/模块/优先级）、新建需求按钮、视图切换
- 支持拖拽变更需求阶段（根据阶段→状态默认映射自动设置对应状态）

### 7.2 需求提交页
- 需求方/PM 填写表单提交新需求
- 必填字段：标题、提出人、所属系统
- 可选字段：brief、描述、PRD链接、墨刀链接、模块、优先级、期望完成时间等

### 7.3 需求详情页
- 展示需求完整信息
- 状态变更操作区（下拉/按钮）
- 评论区
- 操作日志时间线
- 编辑按钮

### 7.4 表格视图
- 类 Excel 表格展示所有需求
- 支持排序、筛选
- 导出为 Excel 文件

### 7.5 后台管理（PM/上级）
- 用户管理（添加/编辑用户）
- 系统/模块配置（管理可选值）

## 8. API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/requirements | 获取需求列表（支持分页/筛选/搜索） |
| POST | /api/requirements | 创建需求 |
| GET | /api/requirements/[id] | 获取需求详情 |
| PUT | /api/requirements/[id] | 更新需求 |
| PATCH | /api/requirements/[id]/status | 变更需求状态 |
| DELETE | /api/requirements/[id] | 删除需求 |
| GET | /api/requirements/export | 导出 Excel |
| GET | /api/comments?reqId=xxx | 获取评论列表 |
| POST | /api/comments | 添加评论 |
| GET | /api/logs?reqId=xxx | 获取操作日志 |
| GET/POST | /api/users | 用户管理 |
